# Contributing to LeadGenius AI

Thank you for your interest in contributing.

## Contribution Workflow

1. Fork the repository.
2. Create a feature branch.
3. Make focused and documented changes.
4. Run lint and test your changes locally.
5. Open a pull request with a clear description.

## Local Development

```bash
npm install
npm run dev
npm run lint
```

## Pull Request Checklist

- Code is readable and minimal.
- Existing behavior is not broken.
- UI updates are responsive on desktop and mobile.
- Documentation is updated when behavior changes.
- No secrets or credentials are committed.

## Coding Standards

- Use TypeScript for type-safe changes.
- Follow existing naming and folder conventions.
- Keep components and functions focused.
- Avoid unrelated formatting-only changes in PRs.

## Issue Reporting

When opening an issue, include:

- Problem summary
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots or logs (if relevant)
- Environment details (OS, Node.js version, browser)
